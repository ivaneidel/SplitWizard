# Batch 12 — Multilanguage UI (English + Spanish)

Dependency-free i18n. Default follows the device language and falls back to English;
the user can override it on the Settings page (System / English / Español), persisted
to localStorage (for instant/offline) and to their profile (cross-device).

## Infrastructure
- **`src/i18n/lang.ts`** — pure language state (no React/Firebase imports, so non-React
  formatters can read it): `LangPref = 'system'|'en'|'es'`, `getStoredPref`, `storePref`,
  `resolveLang` (system → `navigator.language`, else forced; fallback `en`), and a
  module-level `getLang()/setCurrentLang()`. All browser-API access is guarded so tests
  and any non-DOM context don't crash.
- **`src/i18n/index.tsx`** — `LocaleProvider` (reads `useAuth`; adopts `profile.language`
  on load; `setPref` writes localStorage + profile), `useT()` → `{ t, lang, pref, setPref }`,
  and `t(key, vars?)` with `{placeholder}` interpolation and `en` fallback. Re-exports
  `getLang`. Wired in `main.tsx` as `AuthProvider > LocaleProvider > App`.
- **`src/i18n/messages.ts`** merges per-namespace dictionaries; each
  `src/i18n/messages/<ns>.ts` exports `{ en, es }`. Namespaces: nav, login, settings,
  dashboard, group, expense, installments, search, activity, charts, importexport
  (~169 keys, en/es parity verified).
- **`src/lib/date.ts`** — `monthYearLabel` now localizes via `getLang()`, plus a new
  `monthKeyLabel('YYYY-MM')` used by the installments forecast. (`formatMoney` keeps its
  `es-AR` number formatting regardless of UI language — intentional.)
- **`UserProfile.language`** added to `src/types.ts`.

## UI
- Every page/component renders text via `const { t } = useT()` and `t('ns.key')`.
- **Settings**: new "Language" toggle (System / English / Español) next to Appearance.
- Not translated (data, not UI): currency codes, category values, member/display names,
  money/date output, `console.*`, route paths.

## Notes
- Switching language re-renders live (context), including month names and nav.
- No Firestore rules change — `language` is a field on `users/{uid}`, already writable
  by its owner.

## Verify & deploy
- `npx tsc --noEmit`, `npm run build`, `npm test -- --run` (44) all clean.
- Settings → Language → Español: the whole UI switches; System follows the device.
- `npm run build && npx firebase deploy --only hosting`.
